¿Qué se ha construido?

Se ha desarrollado un pipeline backend completo en Python que recibe los assets de un cliente (logo, brandbook PDF, URL corporativa y texto del galardón) y, usando la IA de Anthropic (Claude), genera automáticamente 3 propuestas de diseño personalizadas para el trofeo. Cada propuesta se renderiza como imagen y se compone sobre la fotografía real del producto, devolviendo 3 mockups JPG listos para presentar al cliente.

El sistema funciona en local y está preparado para conectarse al servidor web de Sustain Awards sin modificar la lógica de negocio: solo cambia el envoltorio HTTP.

✓

El código es production-ready en su lógica core. Lo que falta no es código — es configuración (API key), fotografías de producto calibradas y decisiones de negocio sobre modelo de costes y calidad.

Cómo funciona (explicación sencilla)

El cliente sube su logo, su manual de marca (PDF) y nos dice el nombre del premiado y el texto del galardón. A partir de ahí, el sistema hace esto automáticamente:

0
Automático

Preparación de assets

Se lee el logo, se extrae el PDF y se visita la web corporativa del cliente para analizar qué colores usa, si tiene gradientes, qué densidad visual tiene su web. Todo se empaqueta en un contexto unificado.

IA
Claude · Llamada 1

Análisis de marca

Claude ve el logo, lee el PDF y analiza la web. Identifica el tono de la marca, los colores principales, si el logo tiene forma geométrica aprovechable, si el estilo es minimalista o cargado, etc.

IA
Claude · Llamada 2

Selección de 3 composiciones

De 8 estilos compositivos disponibles (gradiente con formas, split cromático, minimalismo tipográfico, patrón radial, etc.), la IA elige los 3 que mejor encajan con esa identidad visual. Siempre garantiza variedad: al menos uno oscuro, uno claro y uno minimalista.

IA
Claude · Llamada 3

Generación de especificaciones de diseño

Para cada una de las 3 composiciones, la IA genera un JSON completo con todos los parámetros del diseño: colores exactos (hex), tipo de fondo, tamaño y posición del logo, color y alineación del texto, formas decorativas, opacidades.

R
Automático

Renderizado de los 3 diseños

Un renderer en Python dibuja cada diseño desde cero siguiendo las especificaciones: genera gradientes de color, coloca las formas decorativas, incrusta el logo del cliente y escribe el texto del galardón con jerarquía tipográfica.

F
Automático

Composición sobre la foto del trofeo

Cada diseño se pega con precisión sobre la fotografía real del trofeo en las coordenadas calibradas de la zona imprimible. El resultado son 3 mockups JPG fotorrealistas que muestran exactamente cómo quedará el producto final.

ℹ

Tiempo total por pedido: entre 15 y 35 segundos. La mayor parte es el tiempo que tarda Claude en analizar y generar los diseños (llamadas a la API). El renderizado en sí es prácticamente instantáneo.

Comparativa de modelos — calidad vs. coste

Claude Haiku 4.5

<$0.01 / pedido

Muy barato. Comprensión de marca limitada. Diseños menos creativos y coherentes. No recomendado para presentar a clientes.

Recomendado

Claude Sonnet 4.6

~$0.03 / pedido

Balance óptimo. Muy buena comprensión visual y creatividad de marca. Calidad profesional. ~12x más barato que Opus.

Claude Opus 4.6

~$0.36 / pedido

Máxima calidad. El mejor análisis de identidad visual. Adecuado para clientes premium o validación del sistema. Actualmente en uso.

Recomendación: usar Sonnet por defecto y Opus para cuentas enterprise o pedidos de alto valor. Con Sonnet, 1.000 pedidos al mes cuestan ~30 € en API. Con Opus, ~360 €. El coste por pedido es marginal respecto al margen del trofeo (precio base desde 30 €).
